# 🐞 BUG-004 – “Too Many Login Attempts” Error Shown on Stripe Link After Single Attempt

---

## 📌 Summary
Stripe Link checkout displays a **“Too many login attempts”** error after a **single login attempt**, even following long inactivity. The user is incorrectly blocked from proceeding with checkout.

---

## 🖥 Environment

| Parameter | Value |
|---------|-------|
| Platform | Web |
| Device | iMac |
| Browser | Google Chrome 144.0.7559.96 (Official Build, arm64) |
| OS | iOS Tahoe 26.2 |
| Reproducibility | Always (100%) |

---

## 🏷 Labels

- Web  
- Payments  
- Stripe  
- Checkout  

---

## 📝 Description

When attempting to subscribe via **Stripe Link**, the system incorrectly shows an error message indicating too many login attempts after **only one email submission**.

This behavior occurs even after long inactivity and without any repeated attempts, resulting in an unjustified lockout from the Link checkout flow.

The error message does not reflect the user’s actual behavior and blocks the payment process.

---

## 🔁 Preconditions

- User is authenticated
- VPN is **not** enabled
- User is on **Pricing & Plan** page
- Stable internet connection

---

## 🧪 Steps to Reproduce

1. Click **“Subscribe Now”** on the Pricing & Plan page
2. Click **“Add payment method”**
3. In the **Secure, fast checkout with Link** section, enter an email address
4. Submit the email (single attempt)

---

## ❌ Actual Result

- System displays error message:  
  **“Too many login attempts for this Link account”**
- User is blocked from Stripe Link checkout despite only one attempt

---

## ✅ Expected Result

- Link checkout should proceed normally after a single attempt
- Error messages should accurately reflect user actions
- If a lock exists, system should display:
  - Cooldown time, or
  - Clear recovery instructions

---

## 📉 Impact

- Users are blocked from completing subscription checkout
- Creates confusion and frustration during payment
- Leads to potential revenue loss
- Reduces trust in checkout reliability

---

## 🔄 Frequency

- Always (100%)

---

## ⚠ Severity

- Medium

---

## 🚦 Priority

- Medium


---

## 📝 Notes

- Issue occurs without multiple login attempts
- Not related to VPN usage or unstable network
- Reproducible after long inactivity

---

## 💡 Recommendation

Review Stripe Link login attempt tracking and cooldown logic to ensure that lockouts are applied only after legitimate multiple attempts.

Improve error messaging to reflect real user actions and provide recovery guidance when applicable.
