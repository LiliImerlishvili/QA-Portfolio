# Regression Testing Workflow

1. Review the changed functionality and affected areas.
2. Identify high-risk and dependent workflows.
3. Select smoke, targeted regression, and full regression coverage.
4. Prepare the required test data and environment.
5. Execute the selected suite.
6. Retest resolved defects.
7. Record passed, failed, blocked, and not-run results.
8. Report remaining risks and release impact.
