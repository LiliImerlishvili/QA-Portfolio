# QA Workflow

## Core Flow

1. Create or open a project.
2. Configure the target and test scope.
3. Run API or security checks.
4. Review findings and reports.
5. Create or update bug reports.
6. Import findings into the test library when needed.
7. Execute linked test cases.
8. Track tasks and retesting.
9. Publish documentation and final QA reports.

## Defect Standard

Each defect should include:

- Clear title
- Test environment
- Preconditions
- Reproduction steps
- Actual result
- Expected result
- Severity and priority
- Evidence
- User or business impact

## Regression Process

1. Review changed areas.
2. Identify impacted modules.
3. Select linked tests.
4. Execute targeted regression.
5. Update findings and tasks.
6. Communicate blockers and residual risks.
