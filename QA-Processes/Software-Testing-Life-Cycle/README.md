# Software Testing Life Cycle

## 1. Requirement Analysis
Review requirements, identify testable conditions, risks, dependencies, and unclear acceptance criteria.

## 2. Test Planning
Define scope, objectives, responsibilities, environments, tools, risks, timelines, and entry and exit criteria.

## 3. Test Design
Prepare test scenarios, test cases, checklists, test data, and traceability.

## 4. Environment Preparation
Confirm the required application build, devices, browsers, accounts, integrations, and access.

## 5. Test Execution
Execute planned and exploratory tests, document results, and report defects.

## 6. Defect Retesting and Regression
Verify fixes and confirm that existing functionality remains stable.

## 7. Test Closure
Summarize coverage, results, unresolved risks, lessons learned, and release recommendations.
