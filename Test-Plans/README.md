# Test Plans

This folder contains project-level test plans defining scope, risks, test approach, environments, entry criteria, exit criteria, and expected QA deliverables.

## Purpose

- Align testing with product risk
- Define what is included and excluded
- Select appropriate test types
- Establish release-readiness criteria
- Clarify responsibilities and dependencies
- Support transparent stakeholder communication

## Common Sections

- Objective
- Scope
- Out of scope
- Risks
- Test approach
- Test environment
- Test data
- Entry criteria
- Exit criteria
- Deliverables
- Reporting process

## Project Collections

- `Mobile-Application-Test-Plan`
- `Quality Engineering Command Center`
- `Vacancy Review and Community Feedback Platform`

## Confidentiality

All test plans are anonymized and exclude confidential requirements, names, URLs, and internal commercial information.
