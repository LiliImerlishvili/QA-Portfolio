# BR — Long Practice Text Moves Beyond the Viewport and Does Not Auto-Scroll

## Summary

After several lines of typing, the remaining practice text moves outside the visible viewport and the screen does not auto-scroll.

## Environment

| Field | Value |
|---|---|
| Environment | Staging |
| Device | iPhone 14 Pro |
| Operating System | iOS 26.4.1 |
| Application Version | 2.0.0 (1.1.12) |
| Reproducibility | 100% |

## Severity

High

## Priority

High

## Preconditions

- The user is on a typing exercise screen.
- A long practice text is available.

## Steps to Reproduce

1. Open the application.
2. Start a practice text containing approximately 50 words.
3. Begin typing the text.
4. Continue for several lines.

## Actual Result

After approximately 5–7 lines, the remaining text is no longer visible inside the viewport. The page does not scroll automatically, forcing the user to scroll manually.

## Expected Result

The active typing line should remain visible. The screen should auto-scroll smoothly or use a fixed layout that keeps the current text and input area visible throughout the exercise.

## User Impact

The user loses visual context and typing rhythm, reducing usability and practice accuracy.
