# Bug Triage Workflow

## Purpose
Review reported defects and agree on severity, priority, ownership, and release impact.

## Workflow

1. Confirm that the issue is reproducible.
2. Review environment and evidence.
3. Check for duplicates or known limitations.
4. Assess user and business impact.
5. Assign severity and priority.
6. Assign ownership.
7. Decide whether the issue blocks release.
8. Track the final decision and next action.
