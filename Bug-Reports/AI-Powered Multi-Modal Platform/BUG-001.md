# 🐞 BUG-001 – Console Warning on Page Refresh (Preloaded Image Not Used)

---

## 📌 Summary
Console warning appears on page refresh indicating that a preloaded image resource is not used shortly after page load.

---

## 🖥 Environment

- **Platform:** Web  
- **Device:** iMac  
- **OS:** macOS Tahoe 26.1  
- **Browser:** Google Chrome 143.0.7499.171 (Official Build, arm64)  
- **Frequency:** 100% (Always)  

---

## 🏷 Labels

- web  
- WEB  

---

## 📝 Description

When refreshing the page, the browser DevTools console displays a warning related to a preloaded image resource that is not used shortly after page load.

Although no JavaScript errors occur, the console is not clean, which violates the expected behavior and may negatively impact performance monitoring and debugging clarity.

---

## 🔁 Preconditions

- Application is accessible  
- User is using Google Chrome browser  
- DevTools Console is enabled  

---

## 🧪 Steps to Reproduce

1. Open the application in a browser  
2. Open **DevTools**  
3. Switch to the **Console** tab  
4. Refresh the page  
5. Observe the console output  

---

## ❌ Actual Result

Console displays the following warning:

> “The resource `https://getbot.ai/assets/logo-secondary.png` was preloaded using link preload but not used within a few seconds from the window’s load event.”

---

## ✅ Expected Result

No console warnings or errors should appear after page refresh.

---

## 🔄 Frequency

- Always (on every page refresh)
