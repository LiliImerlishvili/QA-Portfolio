# 🐞 BUG-010 – ‘Blog post’ Sub-menu Does Not Display Fully and Overlaps Main Prompts Menu

---

## 📌 Summary
The **‘Blog post’ sub-menu** does not render fully after selection and **overlaps the main Prompts menu**, causing some menu items to be hidden and making the UI difficult to use.

---

## 🖥 Environment

| Parameter | Value |
|---------|------|
| Platform | Web |
| Device | iMac |
| OS | macOS Tahoe 26.1 |
| Browser | Google Chrome 142.0.7444.177 (Official Build, arm64) |
| Reproducibility | Always (100%) |

---

## 🏷 Labels

- Web  
- UI/UX  
- Navigation  
- Dropdown / Menu  
- High Priority  

---

## 🔁 Preconditions

- User is **logged in**
- User is on the **Chat Dashboard**
- Prompts menu is accessible

---

## 🧪 Steps to Reproduce

1. Open **GetBot.ai**
2. Click on the **“Prompt”** icon
3. From the dropdown list, click on **“Blog post”**
4. Observe the opened sub-menu

---

## ❌ Actual Result

- The **‘Blog post’ sub-menu is not fully visible**
- Sub-menu **overlaps the main Prompts menu**
- Some menu items are **hidden behind the main menu**
- UI appears visually broken and misaligned

---

## ✅ Expected Result

- The **‘Blog post’ sub-menu** should be fully visible
- Sub-menu should be **properly aligned**
- It should **not overlap or hide** the main Prompts menu
- All options should be clearly accessible

---

## 📉 Impact

- Users cannot see or access all sub-menu options
- Degrades navigation and prompt selection experience
- Causes confusion and reduces usability
- Negatively impacts first-time user experience

---

## 🔄 Frequency

- Always (100%)

---

## ⚠ Severity

- Medium

---

## 🚦 Priority

- High

---

## 📝 Notes

- Issue is reproducible on Chrome
- Likely related to CSS positioning or z-index handling
- Occurs consistently when opening the ‘Blog post’ sub-menu

---

## 💡 Recommendation

Review sub-menu positioning logic and **z-index hierarchy**.  
Ensure the sub-menu expands within available viewport space or adjusts dynamically to avoid overlapping the main menu.
