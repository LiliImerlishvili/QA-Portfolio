# 🐞 BUG-009 – GetBotAI Shows Two Different Versions of the Same Extension UI

---

## 📌 Summary
GetBotAI browser extension displays **two different UI versions simultaneously** in Microsoft Edge:  
one as a **side panel** and another as an **overlay inside the webpage**, causing user confusion and inconsistent behavior.

---

## 🖥 Environment

| Parameter | Value |
|---------|------|
| Platform | Web |
| Device | iMac |
| OS | macOS Tahoe 26.1 |
| Browser | Microsoft Edge 142.0.3595.94 (Official Build, arm64) |
| Reproducibility | Always (100%) |

---

## 🏷 Labels

- Web  
- Browser Extension  
- UI/UX  
- Microsoft Edge  
- High Priority  

---

## 🔁 Preconditions

- User has **GetBotAI extension installed and enabled**
- Browser used is **Microsoft Edge**
- User is logged into the system (if authentication is required)

---

## 🧪 Steps to Reproduce

1. Open **Microsoft Edge**
2. Observe the **GetBotAI extension icon** in the browser
3. Click the **first GetBotAI extension icon**
4. Click the **second GetBotAI extension icon**
5. Observe the UI opening on the **right side panel**
6. Open any website
7. Click the **GetBotAI icon displayed in the center of the webpage**
8. Observe the UI behavior

---

## ❌ Actual Result

- GetBotAI shows **two different versions** of the same extension UI:
  - One opens in the **Side Panel**
  - Another appears as an **Overlay inside the webpage**
- UI behavior is inconsistent
- Users are not informed why multiple interfaces exist

---

## ✅ Expected Result

- Only **one extension interface** should be displayed at a time  
**OR**
- The behavior should be **clearly explained** to the user (Side Panel vs Overlay usage)
- UI behavior should be consistent and predictable

---

## 📉 Impact

- Users may believe the extension is duplicated or broken
- Causes confusion about which interface to use
- Reduces trust in the extension’s stability and UX quality
- Impacts onboarding and first-time user experience

---

## 🔄 Frequency

- Always (100%)

---

## ⚠ Severity

- Medium

---

## 🚦 Priority

- High

---

## 📝 Notes

- Issue is specific to **Microsoft Edge**
- Occurs regardless of visited website
- Both UIs appear functional but lack behavioral clarity

---

## 💡 Recommendation

Unify the extension UI behavior into a **single interface**  
or introduce clear UX guidance explaining when and why each UI (Side Panel vs Overlay) is used.

Consider adding onboarding hints or tooltips to reduce confusion.
