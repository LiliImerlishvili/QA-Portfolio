# Defect Life Cycle

This document describes the main stages a defect passes through from identification to closure.

## Defect Statuses

- New
- Assigned
- In Progress
- Fixed
- Ready for Retest
- Reopened
- Closed
- Rejected
- Duplicate
- Deferred

## Defect Flow

1. QA identifies and reports the defect
2. The defect is reviewed and assigned
3. The developer investigates and fixes the issue
4. QA performs retesting
5. The defect is closed if the fix is successful
6. The defect is reopened if the issue still exists

## Severity and Priority

- Severity describes how seriously the defect affects the system
- Priority describes how urgently the defect should be fixed
- High severity does not always mean high priority
- Priority is usually decided based on business impact and release needs
