# 🐞 BUG-008 – Text Extraction Fails with Network Error for Deepseek Reasoner Model

---

## 📌 Summary
Text extraction from uploaded images fails when using the **Deepseek Reasoner** model.  
The system displays a **network error** and does not return any extracted text, despite successful image upload.

---

## 🖥 Environment

| Parameter | Value |
|---------|------|
| Platform | Web |
| Device | iMac |
| Browser | Google Chrome |
| OS | iOS Tahoe 26.2 |
| Reproducibility | Always (100%) |

---

## 🏷 Labels

- Web  
- AI Vision / OCR  
- Deepseek Reasoner  
- Text Extraction  
- Medium Priority  

---

## 🔁 Preconditions

- User is authenticated (logged in)
- User has an active **Silver subscription**
- User has sufficient **chat credits**
- User has available **image upload limit**
- User is on the **Chat Dashboard**
- **Deepseek Reasoner** model is selected

---

## 🧪 Steps to Reproduce

1. Click on the **Image Upload** icon
2. Select an image containing clearly visible text  
   - Example: `566590_1.jpg` (JPEG, 131 KB, 580×346 px)
3. Click **Crop Image**
4. Click **Upload All**
5. Wait until the image upload completes successfully
6. Click the **Extract Text** button

---

## ❌ Actual Result

- The system does **not extract text** from the image
- A **network error** message is displayed to the user:
- No extracted text is shown in the UI

---

## ✅ Expected Result

- The system should process the uploaded image successfully
- All visible text from the image should be extracted
- Extracted text should be displayed in a text box or modal
- No errors should occur during the extraction process
- User should be able to copy or reuse the extracted text

---

## 📉 Impact

- Core OCR / text extraction functionality is unusable for this model
- Users cannot extract text from images when Deepseek Reasoner is selected
- Feature appears broken and unreliable
- Reduces trust in AI-assisted document processing

---

## 🔄 Frequency

- Always (100%)

---

## ⚠ Severity

- Medium

---

## 🚦 Priority

- Medium

---

## 📝 Notes

- Image upload completes successfully before extraction
- Issue occurs consistently for Deepseek Reasoner model
- Other models may not be affected (requires comparison testing)

---

## 💡 Recommendation

Investigate the network request triggered by **Extract Text** for the Deepseek Reasoner model.  
Ensure proper backend response handling and graceful error messaging if extraction fails.

Fallback behavior or retry logic should be considered to avoid blocking the user flow.
