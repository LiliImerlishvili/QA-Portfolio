# Regression Testing Strategy

## Objective
Confirm that new changes do not break existing functionality.

## Selection Criteria
Include:
- Changed features
- Direct dependencies
- High-risk workflows
- Previously defective areas
- Core smoke scenarios
- Critical integrations

## Execution Levels
- Smoke testing
- Targeted regression
- Full regression

## Reporting
Record passed, failed, blocked, and not-run tests together with unresolved defects and release risks.
